Kernproblem: Es gibt keine zentrale, gut dokumentierte Produkt-/Preislogik. Profil-IDs, Öffnungsarten, Glasgruppen und Preisraster liegen verteilt in JSON, API-Antworten und gebündeltem Frontend-Code. Mehrere IDs waren veraltet oder shop-spezifisch verschieden.

5. Bekannte Risiken / offene Punkte

  • DFS 2-flg/Stulp: Noch nicht implementiert. Große Maße wie 2000×2000 sollten später als 2-flg/Stulp abgebildet werden.
  • Außenöffner: Iglo EXT/Premier haben eigene Öffnungslogik; nicht mit Standard-DK mischen.
  • Farbvarianten: V1 nutzt weiß/weiß. Außen Anthrazit/innen weiß braucht eigene validierte Mapping-Regeln je Anbieter.
  • Zubehör: Griffe, warme Kante, Abstandhalter, Sicherheit, Fensterbankanschluss etc. sind noch nicht Teil des Vergleichs.
  • Fensterblick Profilauswahl: Manche Profil-IDs können per API unerwartet auf ein anderes Profil fallen. Deshalb muss das zurückgegebene Profil-Label immer validiert werden.
  • DFS Rastermaße: DFS nutzt Raster. Für saubere Vergleiche entweder Rastermaße als Benchmark wählen oder Maße als „gerundet“ ausweisen.

Ricky's Empfehlung

  1. Eine zentrale Mapping-Tabelle pflegen: interne DFS-Profil-ID, Fensterblick-ID, Fensterversand-ID, Status, Label-Regex, letzte Prüfung.
  2. Nur validierte Profile live schalten: Profil gilt erst als vergleichbar, wenn API-Label + Maße + Preis > 0 stimmen.
  3. Benchmark bewusst klein halten: 2 Standardgrößen pro Profil + 2fach/3fach reichen für Marktposition. Details über Live-Konfigurator.
  4. Preislogik dokumentieren: Basispreis, Glas, Farbe, Öffnung, Zubehör, Marge/MwSt. als eigene Module und Tests abbilden.
  5. Regressionstest wöchentlich: Cron sollte nicht nur Preise ziehen, sondern auch prüfen: Profil-Label stimmt, Maße stimmen, keine 0-Preise, keine Silent-Clamps.
  6. 2-flg/Stulp als nächster Sprint: Erst danach große Maße aufnehmen. Sonst entstehen falsche Vergleiche.
  7. Admin-Seite „Mapping Audit“ behalten: Fehlende Profile nicht verstecken, sondern intern sichtbar machen: „nicht im Angebot“, „Mapping fehlt“, „Sonderlogik nötig“.
